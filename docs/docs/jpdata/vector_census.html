4. ベクタデータ (国勢調査)

べクタデータのうち、国勢調査に特有の機能について記します。

4.1. ダウンロード

国勢調査のダウンロードについては、小地域集計とメッシュの双方に対応しています。

なお、原則として座標参照系は、JGD2011 の平面直角座標系を選択するようにしています。 古いデータでは JGD 2000 のこともあります。

QGIS を立ち上げて、最初のデータとして国勢調査の地図を追加すると、地理座標系ではなく、単位をメートルとした座標系となります。

なお、一般的には、平面直角座標系は地域で区分されているため、例えば東京都の第９系ですが、小笠原諸島は第１８系となっています。 しかし、国勢調査に限っては小笠原村も第９系となっています。 このように、都道府県単位で系番号が統一されているようです。

4.2. 地図に追加とテーブル結合

テーブル結合は、GIS で二つのレイヤをの属性テーブルを結合する技術のことです。

国勢調査では、属性が膨大にあるため、基本的な部分のみシェープファイルに組み込まれています。 他の属性は、CSV 形式で提供されています。

また、メッシュについては、基本的な属性は GIS データに入っておらず、すべてCSV 形式の別ファイルとなっています。

年度によって異なりますが、2020 年の場合は、以下のような属性テーブルがあります。

小地域集計 (2020年度)

  • 男女別人口総数及び世帯総数 シェープファイルにもある

  • 年齢（５歳階級、４区分）別、男女別人口 jpData では、自動的にテーブル結合

  • 世帯人員別一般世帯数

  • 世帯の家族類型別一般世帯数

  • 住宅の所有の関係別一般世帯数

  • 住宅の建て方別世帯数

  • 産業（大分類）別及び従業上の地位別就業者数

  • 職業（大分類）別就業者数

  • 世帯の経済構成別一般世帯数

メッシュ (2020年度)

  • 人口及び世帯 jpData では、自動的にテーブル結合

  • 人口移動、就業状態等及び従業地・通学地

  • ５歳階級別人口

  • 労働力状態、産業分類及び職業分類別人口（１５歳以上）

  • 住宅の所有及び建て方

  • ５年前の常住地及び従業地・通学地等

メモがないテーブルは、現状は未対応です。 プログラム的には準備はできていますが、どのようなユーザーインターフェイスが良いかが決まっていません。 開発に協力していただける方を待っています。

4.2.1. 国勢調査の CSV

国勢調査のCSVファイルは、「機械判読」がしづらいフォーマットです。 jpData では、これを機械判読しやすい CSV 形式に変化してからテーブル結合します。

4.3. ソースコード

4.3.1. ソースコード: テーブル結合

2 行目の文字列は、PROJ 文字列(Proj String) と言います。 aeqd は、正距方位図法を指しています。

1   from qgis.core import QgsCoordinateReferenceSystem, QgsProject
2   proj_string = "+proj=aeqd +lat_0=139.691722 +lon_0=35.689501 +datum=WGS84 +units=m +no_defs"
3   crs = QgsCoordinateReferenceSystem.fromProj(proj_string)
4   if crs.isValid():
5      QgsProject.instance().setCrs(crs)
